# 🗂️ KanbanApp
**An offline‑first kanban app for desktop, using local storage.**
***
## 🧭 About this project
The goal of this project was to create a simple offline desktop application with local storage. I built it to strengthen my practical skills in JavaScript, HTML and CSS, to practise rapid development, and to experiment with using these familiar tools inside an Electron environment. I also wanted to see what I could build for myself and how far I could take the idea with a straightforward setup.
***
## 🛠️ What this does
This app uses the visual elements of kanban, including boards, columns, and cards, to help you organise tasks. Trello is a popular tool for this, as a result I tried to create something similar with an offline-first, local storage approach. 

**App functionality:**
- Create, read, update and delete (CRUD) boards, columns and cards.
- Drag and drop cards between columns.
- Create colourful labels cards.
- Add due date/time to cards, that changes colour depending on when the task is due.
- Add descriptions and notes to cards.
- Mark cards as complete.
- Icons show on cards to indicate what data it holds. e.g. description, date etc.
- Add a background image to each board with optional blur effect.
- Saves data locally to JSON file.
- Change the accent colour of the app.
- See how many cards are currently in a column.
***
## 🔑 Key Takeaways
- I strengthened my practical understanding of JavaScript, HTML and CSS by applying them in a desktop‑app context rather than a browser.
- I gained hands‑on experience with Electron and learned how its behaviour differs between development and production.
- I discovered that Electron introduced some bugs and inefficiencies I didn’t expect, which helped me understand its limitations and why it may not have been the right tool.
- I improved my ability to prototype, test features, and iterate quickly.
- I learned how to structure a small project for speed and simplicity, and what I would change if I were building something more maintainable.
- I really enjoyed working with CSS and front‑end elements, especially when the visual feedback is immediate.
